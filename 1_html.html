Cuando el negocio de impresiones no usaba computadoras, se hacían anotaciones que indicaban la forma de imprimir cada parte del texto.

Hoy en día, las indicaciones de impresión se mezclan con el texto y reciben el nombre de marcas o etiquetas, de las cuales hay varios tipos:

Etiquetas iniciales

Marcan el inicio de una anotación. Son de la forma

<marca atributo1="valor1" atributo2="valor2">

donde atributo1 y atributo2 son los nombres de las características de la etiqueta; por ejemplo altura o margen, mientras que valor1 y valor2 son los valores de que tienen los atributos, por ejemplo 20 puntos o 1 línea arriba y abajo.

Algunos atributos son boolean y solo se pone su nombre, sin indicar el valor, para representar true; por el contrario, si no se ponen en la etiqueta inicial, valen false. Como ejemplo están los atributos readonly o hidden.

Ejemplo:

<input type="text" value="Hola" readonly>

Etiquetas de cierre

Marcan el final de una anotación. Son de la forma

</marca>

Comentarios

Inician con <!--, terminan con -->, pueden abarcar uno o más renglones y son ignorados por los navegadores web.

Ayudan a entender como funciona el código. Una limitación es que no pueden contener la secuencia --.

Ejemplo:

<!-- Este es un comentario. -->

En HTML hay etiquetas predefinidas para diferentes partes de un texto, con instrucciones de impresión también predefinidas. Por ejemplo, h1 para encabezados y p para párrafos.

El ejemplo con las instrucciones de marcado, quedaría así:

<h1>Primer Ejemplo</h1>
<p>Hola Mundo</p>

Ya tenemos el concepto de marca, pero falta explicar de forma más precisa el contenido de un archivo HTML.

El código HTML requiere usar etiquetas adicionales para indicar las características de un documento.

Las marcas de los archivos HTML se utilizan para representar objetos llamados elementos.

Cada elemento tiene la siguiente estructura interna:

Tipo de Elemento

Indica el tipo de información que representa el objeto. Por ejemplo, p representa un párrafo o h1 representa un encabezado de primer nivel.

Atributo

Características a las cuales se puede asignar un valor. Por ejemplo:

  • maxsize="10"
  • id="texto1"
  • height="150"

Sirven para configurar la forma en que se muestra un elemento. Son opcionales y no hay límite para la cantidad de atributos que puede contener un elemento.

Identificador

Es un atributo opcional llamado id. Todos los elementos pueden tener este atributo, pero para cada elemento debe tener un valor diferente; no se vale que se repita.

Es el nombre que distingue a un elemento de todos los otros elementos del archivo.

Contenido

Es lo que va en el interior del elemento y puede llevar:

  • Texto que forma parte del elemento.
  • Comentarios.
  • Más elementos que complementan al elemento.

El contenido HTML consta de las siguientes partes:

Comentarios

Inician con <!--, terminan con -->, pueden abarcar uno o más renglones y son ignorados por los navegadores web.

Ayudan a entender como funciona el código. Una limitación es que no pueden contener la secuencia --.

Ejemplo:

<!-- Este es un comentario. -->

Elementos con Contenido

Llevan:

  • Una etiqueta inicial
  • Contenido
  • Una etiqueta de cierre

Tienen una estructura como la siguiente:

<tipo_de_elemento atributo1="valor1" atributo2="valor2" atributo_booleano1 atributo_booleano2>
  Contenido HTML
</tipo_de_elemento>

Elementos sin Contenido

Solo usan una etiqueta inicial y tienen una estructura como la siguiente:

<tipo_de_elemento atributo1="valor1" atributo2="valor2" atributo_booleano1 atributo_booleano2>

Texto
El texto que forma parte del elemento.

Un archivo de HTML5 debe tener al menos la siguiente estructura:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título</title>
  </head>
  <body>
    <!-- Aquí va el contenido que muestra el navegador web. -->
  </body>
</html>
<!DOCTYPE html>

Se conoce como DOCTYPE e indica al navegador la estructura y comportamiento que deben tener las etiquetas.

El valor mostrado es obligatorio para HTML5 o posterior.

<html lang="es">

El elemento html es el elemento principal del archivo. Solo puede se puede tener una de estas etiquetas en un archivo.

También hay que indicar el código de idioma en el atributo lang, que en este caso corresponde al español.

<head>

Es el primer elemento dentro de html y contiene información para procesar el archivo.

<meta charset="UTF-8">
meta

Indica metadatos, o sea información sobre el archivo.

charset="UTF-8"

Indica que el archivo codifica los caracteres del archivo usando el estandar Unicode, que sirve para casi todos los idiomas del mundo.

El lenguaje JavaScript usa por defecto Unicode, por lo que no poner esta etiqueta, podría ocasionar problemas con los elementos script de la página.

Si no se pone esta etiqueta, se usa el estandar ISO-8859-1, que sirve solo para los paises de Europa Occidental y América.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
meta

Indica metadatos, o sea información sobre el archivo.

name="viewport"

Indica la definición del metadato viewport, que indica como manejar la pantalla en navegadores de dispositivos móviles.

content="width=device-width, initial-scale=1.0"

Es el valor asociado con el metadato viewport, que consta de varios atributos.

width=device-width

El ancho de la página será el ancho del dispositivo, lo que permite mostrar las letras del tamaño normal para el móvil.

initial-scale=1.0

Inicialmente, la página se muestra sin modificar el tamaño natural de la página. Es decir que ni se amplifica ni se reduce el contenido.

Si no se pone esta etiqueta y la página se muestra en un navegador móvil, se desplegará con letras reducidas en tamaño.

<title>Título</title>

El contenido del elemento title se muestra en las pestañas o en las barras de título de las ventanas que muestran la página.

<body>

El contenido del elemento body es el que se muestra en el navegador web.

Representa el texto de un documento y los elementos que lo modifican.

No implica ninguna jerarquía ni estructura.

Se puede colocar dentro de otros elementos que si implican estructura; por ejemplo, párrafos, títulos, tablas, etcétera.

Los elementos h1, h2, h3, h4, h5 y h6

Representan títulos de secciones.

Su contenido de fraseo.

Los elementos tienen una categoría en base a su número, donde h1 es la categoría más alta y h6 es la más baja.

Títulos de una misma categoría o mayor definen nuevas secciones.

Títulos de una categoría más baja definen subsecciones que son parte de la anterior.

Se recomienda que haya un solo h1 en una página y sea el título del documento. Las secciones se pueden marcar con h2 o inferior.

Es un elemento de fraseo.

Se le conoce como ancla.

Cuando un elemento tiene el atributo href, representa un hipervínculo. El texto que muestra es su contenido, que debe ser de fraseo.

Ejemplo:

<a href=" https://google.com ">Google</a>

Atributos

href

URL válida del recurso que muestra. Puede estar rodeado por espacios.

Representa el pie de la página o de una sección.

Típicamente contiene información de quien escribió la página o sección, enlaces a documentos relacionados, copyright y cosas similares.

Cuando contiene secciones, representan apéndices, índices, acuerdos de licencia y contenidos similares.

Es un elemento que conceptualmente puede llevar anclas.
