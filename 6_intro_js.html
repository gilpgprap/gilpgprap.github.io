Un identificador es el nombre que tiene un elemento de un programa y que no se puede repetir en ciertas partes del código.

Reglas que Debe Cumplir un Identificador

  • Empezar con una letra (a – z o A – Z), guion bajo (_) o $.

  • Los caracteres siguientes pueden ser cualquiera de los anteriores o dígitos (0 – 9).

Ejemplos

  • _nombre
  • __nombre
  • Nombre24
  • _
  • $
  • n123k_$K
  • TaChido
  • serenoMoreno
  • $priv
  • $123
  • h54

Camel Case

Aunque no es obligatorio, normalmente en JavaScript los identificadores se escriben en Camel Case, donde las mayúsculas se usan para simular el inicio de una palabra.

Ejemplo:
estasSonCuatroPalabras.

Muestra una serie de valores.

Aunque al escribir los el código se escriben separados por coma, se muestran separados por espacio.

Cada instrucción console.log es un renglón.

Ejemplo

console.log("Hola", "pp");
console.log("tt");

Los comentarios son secciones de código que no son procesadas y explican otras partes del programa para que se puedan entender mejor.

Tipos de Comentarios

Comentarios de una Línea

Empiezan con el texto // y terminan con el final de la línea.

Normalmente se colocan en la línea inmediatamente antes del texto que explican, aunque también se colocan al final de esta.

Ejemplos:

  • // Crea la variable a y le asigna 0.
    let a = 0;
  • let b = 0; // Crea la variable b y le asigna 0.

Comentarios de una o más Líneas

Empiezan con el texto /* y terminan con */. Pueden usar una o más líneas.

Se colocan en la línea inmediatamente antes del texto que explican, aunque también se colocan al final de esta.

Ejemplos:

  • /* Crea la variable c y le asigna 0. */
    let c = 0;
  • /* Crea la variable d y
    le asigna el valor numérico 0. */
    let d = 0;
Comentarios de Documentación

Son comentarios de una o más líneas que empiezan con /** y explican la definición de un elemento del código.

Se colocan en la línea inmediatamente antes del texto que explican.

El contendo de este tipo de comentarios debe seguir las reglas de JSDoc o de revisión de TypeScript para archivos JavaScript.

Ejemplo:

/** Variable numérica cuyo valor inicial es 0.
 * @type {number} */
let e = 0;

/* Muestra "Hola pp". */
console.log("Hola", "pp");
// Muestra "tt".
console.log("tt");
console.log("bb"); // Muestra "bb".

Se declaran con let o con var, aunque se recomienda usar let.

Tienen asociado un nombre, que tiene la estructura de un identificador y normalmente empieza con minúscula.

Tienen asociado un valor que puede cambiar durante la ejecución del programa.

Ejemplo

let a = 1;
console.log(a);
console.log("a");
a++;
console.log(a);

Similares a las variables, pero su valor no puede cambiar una vez asignado.

Se declaran con const

Ejemplo

const PI = 3.14159;
console.log(PI);
PI++;
console.log(PI);

Salida

TypeError: Assignment to constant variable.

Agrupan 0 o más sentencias, delimitadas por un par de llaves.

Las variables declaradas con let o const dentro de un bloque son diferentes de las variables o constantes con el mismo nombre en el exterior del bloque.

Ejemplo

const X = 3;
console.log(X);
{
  const X = 4;
  console.log(X);  
}
console.log(X);
